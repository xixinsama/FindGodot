## 游戏设计

在网格上根据规则寻找隐藏的godot

最复杂情况：N*M网格上，有不同颜色区域但不显示，横列和竖列会显示该颜色的数量（数织），需要根据数字信息推理颜色区域分布，颜色区域必定相邻（不包括斜角，即必定上下左右相邻）。每种颜色内只有且仅有一只godot，每个godot周围8格内没有其他godot，每个godot的同一行和同一列的格子里都没有其他godot

最简单情况：N*M网格上，显示几个godot；godot的同一行，同一列和斜角直线上里都没有其他godot，即《N皇后问题》；玩家需要找到其余的godot，可能有多个解

由难度梯度拆分出来控制选项：

1. 数织
2. 颜色区域
3. godot互斥规则（网格）

不开数织：N*M网格上，有不同颜色区域，颜色区域必定相邻（不包括斜角，即必定上下左右相邻）。每种颜色内只有且仅有一只godot；每个godot周围8格内没有其他godot，每个godot的同一行和同一列的格子里都没有其他godot

不开颜色：此时相当于少了一条godot互斥规则：
**“每种颜色内只有且仅有一只godot”**。
缺少规则意味着开局缺少信息，意味着在最复杂情况下，可以一个godot都不显示的情况下开，在这种规则下必须显示几个godot来减少解的数量，或是单纯支持多解


互斥规则拆解：

* 区域：区域（颜色）内有且仅有一只godot
* godot范围：叫godot攻击范围比较贴切，攻击范围内不允许有其他godot
* 对偶：以上两个的结合，在godot的某个范围内，必然有一个别的godot

注意到横向或竖向的互斥规则和数织规则是相似的，所以我打算将其作为底层逻辑


规则：

1. godot的同一行，同一列以及周围8格内都没有其他godot（必选）
2. 旋转，将攻击范围旋转45°；即变为：godot的斜角直线上没有其他godot（开关）
4. 颜色区域：颜色区域内有且仅有一只godot(必选)
5. 数织：颜色区域改为抽象符号，不直接显示在网格上（if规则4开，则显示，否则不显示；开关）
